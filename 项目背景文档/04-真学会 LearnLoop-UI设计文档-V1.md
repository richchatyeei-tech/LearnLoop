## 1. 设计目标与原则

### 1.1 一句话设计目标

让老师在「效率与决策」中信任它，让学生在「鼓励与陪伴」中愿意用它，全程不出现任何系统术语。

### 1.2 三条总原则

1. **转译优先**：界面只出现用户语言（确认布置订正 / 学情回收 / 学会率），不出现内部术语（交接点 / fan-out / Loop / 掌握矩阵 / 阈值）。
2. **场景分组**：功能按用户工作时间线组织（作业准备 → 批改 → 讲评 → 回收 → 沉淀），不按数据流或技术模块组织。
3. **两端异质**：教师端重信息密度与决策；学生端重专注、鼓励与即时反馈。

### 1.3 两端体验差异

| 维度 | 教师端（PC） | 学生端（Pad） |
| --- | --- | --- |
| 信息密度 | 高，一屏看全班 | 低，一次只专注一道题 |
| 语气 | 专业、效率、可信 | 鼓励、亲切、轻松 |
| 视觉重心 | 数据 / 列表 / 决策按钮 | 大字 / 图 / 即时正反馈 |
| 决策权 | 集中在老师（确认才分发） | 跟随引导，只需「试一试」 |

## 2. 设计系统（Design Tokens）

### 2.1 色彩

整体为「暖纸色 + 墨绿」体系，刻意避开通用的紫蓝渐变 AI 风，传达教育的温度与可信。

| Token | 值 | 用途 |
| --- | --- | --- |
| `--paper` | `#efeae0` | 页面底色（暖纸） |
| `--card` | `#fbfaf6` | 卡片底色 |
| `--card-2` | `#f4f1e9` | 次级底色 / 轨道 |
| `--ink` | `#211f1b` | 主文字 / 强调按钮 |
| `--ink-soft` | `#6f6a60` | 次级文字 |
| `--ink-faint` | `#a39d90` | 占位 / 辅助 |
| `--line` | `rgba(33,31,27,.10)` | 分割线 / 描边 |
| `--brand` | `#1c6b4f` | 品牌墨绿（学会 / 闭环关闭） |
| `--brand-deep` | `#134d39` | 深墨绿（文字强调） |
| `--brand-soft` | `#d6e7df` | 墨绿浅底 |
| `#6ad0a6` | 亮绿 | 主操作按钮 / 进度高亮 |

**状态语义色**（贯穿两端，禁止改色）：

| 含义 | 色 | Token |
| --- | --- | --- |
| 学会了 / 正向 | 墨绿 | `--brand` |
| 还没学牢 / 提醒 | 琥珀 | `--amber #c8821f` |
| 建议面批 / 错误 | 砖红 | `--red #b8442f` |

### 2.2 字体

| 角色 | 字体 | 用途 |
| --- | --- | --- |
| 展示 / 品牌 | Bricolage Grotesque | 大标题、品牌名、关键数字 |
| 等宽 | JetBrains Mono | 英文标签、编号、指标数值 |
| 正文（中文） | PingFang SC / 系统 | 所有正文 |

原则：大标题与关键数字用展示字体制造记忆点；正文克制，保证可读。

### 2.3 形状 / 阴影 / 间距

| Token | 值 |
| --- | --- |
| 卡片圆角 `--r` | `16px`；Pad 设备框 `34px`；小元素 `8–12px` |
| 阴影 `--shadow` | `0 1px 2px /.06 + 0 8px 24px /.06` |
| 浮起 `--shadow-lg` | `0 4px 12px /.08 + 0 24px 60px /.12` |
| 页面内边距 | 教师主区 `24–30px`；卡片 `16–20px` |

### 2.4 品牌符号：Loop 环

LearnLoop 的标识是一个未闭合的圆环 + 中心点：外环墨绿（代表闭环推进），中心琥珀点（代表一次作业 / 错题原子）。用于 Logo、学生端进度环、掌握度环，是全局的视觉母题。

### 2.5 图标

- 概念原型阶段使用 Emoji（🏠 📝 📥 📈 💬 等）以快速表意。
- **生产化建议**：替换为统一 SVG 图标集（Lucide / Heroicons，24×24 viewBox），避免 Emoji 跨平台渲染不一致。状态色仍由上方语义色驱动，颜色不可作为唯一区分手段（需配文字 / 形状）。

## 3. 全局框架

### 3.1 教师端（PC Web）

```text
┌───────────────────────────────────────────────┐
│ 顶栏：品牌 + 标语 + 班级切换 + 演示视图切换       │
├──────────┬────────────────────────────────────┤
│ 侧边栏    │  主内容区（按模块切换）               │
│ 教学组    │                                     │
│  工作台   │                                     │
│  作业布置 │                                     │
│  作业管理 │  ← 总台账（全部作业+全部状态）        │
│  智能批改 │  ┐ 各自「列表(按阶段过滤) → 详情」     │
│  备课讲评 │  ├ 列表只列处于本阶段的作业           │
│  学情回收 │  ┘ 详情顶部统一「生命周期流转条」      │
│ 沉淀组    │                                     │
│  题库     │  ← 题目资产沉淀（入库/变式/复用）      │
│  学情溯源 │                                     │
│  家长摘要 │                                     │
│ 个人信息  │                                     │
└──────────┴────────────────────────────────────┘
```

- 侧边栏宽 `224px`，按「教学 / 沉淀 / 班级与设置」分组；当前项墨色高亮；待办项右侧红色数字角标。
- **题库归属「沉淀」组**：题目是跨 Loop 复用的内容资产（入库、变式、布置引用），不属于单次教学动作链路；与学情溯源、家长摘要同属「沉淀与复用」层。
- 主内容区 `max-width 1180px`，每个模块一个 page，切换带淡入。
- **顶栏含班级切换**：本期默认初二(3)班，另挂 1–2 个班作上下文，体现「多班级」维度。
- **统一「列表 → 详情」模型**：作业管理、智能批改、备课讲评、学情回收四个阶段页**结构一致**，均为「列表页 → 详情页」两段式（`subview` 切换，详情页带「← 返回列表」）。
  - **作业管理 = 总台账**：跨班级 / 作业次列出**全部**作业 + 生命周期状态 pill；点某条作业，路由到其当前阶段的详情。
  - **智能批改 / 备课讲评 / 学情回收 = 阶段工作台**：各自列表只列**处于本阶段**的作业（按 PRD §9.1 状态过滤），分「待处理（本阶段） / 已流转（已进入下一阶段）」两组；点「待处理」进本阶段详情，点「已流转」深链接到其当前阶段详情，状态 pill 始终反映**当前真实状态**。
- **详情页统一「生命周期流转条」（lifeflow）**：每个阶段详情页顶部固定一条 5 节点流转条「**作业管理** → 智能批改 → 备课讲评 → 订正回收 → 已闭环」，已完成节点打勾、当前节点高亮、后续节点置灰，让老师在任一页都能一眼看清「这份作业到哪一步、上一步是谁、下一步交给谁」（状态与关系，对齐 PRD §9.1）。

### 3.2 学生端（Pad Web）

## 4. 页面规范（对齐 PRD 模块）

> 实现状态：✅ 原型已实现 / 🟡 待接 API（工程联调阶段）

### 4.1 学情溯源页面详述（三层下钻）

学情溯源是唯一的跨作业 / 跨时间视图，采用 **L1 全部班级 → L2 单个班级 → L3 学生个人** 的下钻漏斗（对应 PRD §7.1）。三层共用顶部一条**维度切换 / 面包屑**（`全部班级 ▸ 初二(3)班 ▸ 张磊`），默认进入 L1，点击逐层下钻，面包屑可逐层返回。三层的分析目的不同，呈现方式也刻意不同：**L1 用"对比"图表、L2 用"趋势 + 下钻"、L3 用"个体画像"**。

**L1 全部班级（对比型）** —— 回答"精力往哪倾斜、是教法还是某班问题"：

| 区块 | 布局 | 呈现 |
| --- | --- | --- |
| L1-A 班级体检榜 | 各班一行：排名 + 班名 + 真掌握率对比条 + 闭环/待提升人数；整行可点下钻 L2 | 横向对比条（rankrow），最弱班用琥珀/红 |
| L1-B 共性 vs 个性弱项 | 知识点 × 班级 矩阵，单元格按掌握度着色 + 判定列（共性/个性/普遍掌握） | 热力矩阵（cmx），全行红=共性、单格红=个性 |
| L1-C 教学有效性跨班对比 | 同一讲评在各班的提升幅度条；底部一句话归因 + 「下钻该班」链接 | 提升幅度条（xrow），无效班用琥珀 |

**L2 单个班级（溯源型）** —— 回答"错什么、根因、讲评有没有用、谁掉队"：

| 区块 | 布局 | 呈现 |
| --- | --- | --- |
| 健康度 KPI | 顶部 4 张：真掌握率(环比)/闭环作业/有效讲评率/结构性弱项 | stat 卡 |
| L2-① 真掌握率趋势 | 整宽：真掌握率 vs 表面正确率双线 + 讲评点竖线 | 折线图（SVG），双线 + 讲评标记 + 图例 |
| L2-② 知识点薄弱地图 | 半宽：8 格知识点热力 + 根因链（根因→下游连环错） | 热力网格（hcell）+ 根因链（chain）|
| L2-③ 教学有效性 | 半宽：各知识点"讲评前→后"双条 + 提升幅度 + 有效/无效标签 | 前后对比条（eff），有效绿/无效红 |
| L2-④ 重点学生 | 整宽：分层条 + 掉队/回升学生及其掌握轨迹序列；点学生下钻 L3 | 分层条（tierbar）+ 学生轨迹（review-list + seq）|

**L3 学生个人（个体型）** —— 把班级问题落到一个人，并接回闭环：

| 区块 | 布局 | 呈现 |
| --- | --- | --- |
| 身份卡 | 顶部画像卡：头像 + 学号/班级/Pad + 当前真掌握率(环比) + 班级平均 + 分层标签 | stu-head 卡 |
| KPI | 4 张：真掌握率/订正闭环率/反复点/班级排名 | stat 卡 |
| L3-① 掌握率轨迹 | 整宽：本人真掌握率 vs 班级平均双线 + 讲评点 | 折线图（SVG），本人红实线 / 班级绿虚线 |
| L3-② 知识点掌握分布 | 半宽：各知识点掌握度条，按掌握度着色 | 分布条（kbar）|
| L3-③ 错因构成 | 半宽：错因类型占比环形图 + 图例 | 环形图（donut）+ 图例 |
| L3-④ 逐次作业明细 | 整宽：每次作业 得分/真掌握率/主要错因/是否订正闭环 | 横向表格（stable）+ 闭环 pill |
| 行动 | 底部：AI 一句话诊断 + 「布置专属补练」「生成家长摘要」 | 建议卡 + action-bar |

> 取舍：L3 完整跑通 1 个掉队学生（张磊）作示例，其余学生点击占位；行动按钮经 Toast 反馈，体现"诊断→接回 fan-out / 家长摘要"。

## 5. 核心组件规范

| 组件 | 说明 | 关键状态 |
| --- | --- | --- |
| 数据卡（stat） | 标签 + 大数字（展示字体）+ 说明 | 数值色随语义（学会绿/没牢琥珀/面批红） |
| 错因条（bar） | 题号 + 题干 + 错误率 + 轨道 + 错因说明 | 高错误率红填充，中等琥珀填充 |
| 讲评列表（review-list） | 序号徽标 + 标题 + 说明 | 序号色：建议=墨绿，提醒=琥珀，警示=红 |
| 学生 chip | 圆点（错因色）+ 姓名 + 错因 | — |
| 掌握矩阵（matrix） | 10 列网格，每格一名学生 | 绿=学会 / 琥珀=没牢 / 红=面批 / 灰=未完成 |
| 确认闸门（action-bar） | 黑色吸底条：说明文案 + 次按钮 + 主按钮 | 主按钮亮绿，hover 微浮起；点击后推进状态 + Toast |
| 进度环 / 掌握度环 | Loop 环母题，墨绿弧 + 数值 | 随步骤/掌握度更新 dashoffset |
| 闯关步骤条 | 四段进度：为什么错/跟着改/换题试试/学会啦 | 当前段亮绿，已完成墨绿 |
| 变式选项（opt） | 字母键 + 选项文字 | 正确=墨绿底，错误=红底；未答禁用「下一步」 |
| 导入条（importer） | 虚线大卡：图标 + 导入说明 + 三种来源分段（题库/上传/手动） | hover 描边墨绿；点击触发导入 Toast |
| 题目卡（qedit） | 题号徽标 + 题干 + 标准答案 pill + 知识点标签（ktag） | 答案 pill 墨绿浅底；知识点标签灰底描边 |
| 提交网格（subgrid） | 8 列网格，每格一名学生提交状态 | 绿=已交 / 灰虚线=未交（配 ✓ / · 与文字，颜色非唯一区分） |
| 批改条（grade-row） | 题号 + 题干 + AI 判分 pill + 错因说明 + 认可/改判按钮 | 错误率 pill 随语义；选中按钮墨绿实底 |
| 班级切换器 | 顶栏下拉/分段：当前班级 + 可切其他班 | 当前班高亮；切换后全局上下文随之刷新 |
| 作业列表行 | 图标 + 作业名 + 班级/作业次/时间 + 状态 pill | 状态 pill 随生命周期（待批改/待确认/待讲评/订正回收中/已闭环）；点「待处理」进本阶段详情，点「已流转」深链接到当前阶段详情 |
| 生命周期流转条（lifeflow） | 阶段详情页顶部：作业管理→智能批改→备课讲评→订正回收→已闭环 五节点 + 连接线 | 已完成节点墨绿打勾；当前节点墨色实心 + 墨绿光圈；后续置灰 |
| 学生得分表 | 批改详情下钻：学生名 + 各题得分 + 总分/错因数 | 行可点击 → 展开单生作答与错因详情面板；低分行琥珀/红提示 |
| 题库选题器 | 按知识点/难度筛选 + 题目勾选入卷 | 已选计数；入卷态高亮（本期预置题库） |
| 拆题确认卡 | 上传图片 → AI 拆出的单题：题干/答案/知识点可逐题编辑确认 | 待确认琥珀，已确认墨绿；可改判知识点（本期预置兜底） |
| 历史聚合条 | 备课讲评顶部：本班跨作业共性弱项 + 趋势一句话 | 文案陈述；可点知识点跳学情溯源 |
| 生成按钮（讲评建议） | 「生成讲评建议」动作按钮，可重新生成 | 生成中 loading；生成后列表可编辑保存 |
| 维度切换 / 面包屑（scope-tabs） | 学情溯源顶部：全部班级 ▸ 班级 ▸ 学生 三层下钻 | 当前层高亮（全部=墨绿、班级=墨色、学生=墨色）；点击切换或返回 |
| 班级体检榜行（rankrow） | L1：排名 + 班名 + 真掌握率对比条 + 闭环/待提升 | 整行可点下钻 L2；最弱班对比条用琥珀/红 |
| 共性/个性矩阵（cmx） | L1：知识点 × 班级 掌握热力 + 判定列 | 单元格四级着色（红/琥珀/浅绿/墨绿）；判定文字标共性/个性 |
| 跨班有效性条（xrow） | L1：同一讲评在各班的提升幅度对比 | 有效绿填充 / 无效琥珀填充 + 幅度数字 |
| 真掌握率趋势（chart） | L2-①/L3-①：双线折线 + 讲评点竖线 | 真掌握墨绿实线 vs 表面正确率/班级平均虚线；讲评点带描边圆点 |
| 知识点热力（hcell）+ 根因链（chain） | L2-②：知识点掌握网格 + 根因→下游链路 | 四级着色；根因节点红底，箭头连下游 |
| 教学有效性前后条（eff） | L2-③：讲评前→后双条 + 提升幅度 + 标签 | 前=琥珀渐变 / 后=墨绿渐变；有效绿标 / 无效红标 |
| 分层条（tierbar） | L2-④：掌握牢 / 基本掌握 / 待提升 三段占比 | 墨绿 / 浅绿 / 琥珀分段，宽度按人数 |
| 学生画像头（stu-head） | L3：头像 + 学号/班级 + 真掌握率(环比) + 班级平均 + 分层标签 | 真掌握率随语义着色；掉队=红标签 |
| 知识点分布条（kbar） | L3-②：各知识点掌握度横条 | 四级着色，弱项靠前展示 |
| 错因环（donut） | L3-③：错因类型占比环形 + 图例 | conic 渐变（审题红/概念琥珀/计算蓝/步骤灰）+ 中心主因 |
| 逐次明细表（stable） | L3-④：每次作业 得分/真掌握率/主要错因/闭环 | 真掌握率随语义；闭环=绿 pill / 未订正=红 pill |
| Toast | 黑底吸底提示，亮绿圆点 | 用于交接点反馈（已布置 / 已回流）与主线流转（已发布 / 已收齐 / 已确认） |

## 6. 交互与动效

| 场景 | 行为 |
| --- | --- |
| 模块 / 页面切换 | 淡入上移 `fade .35s` |
| 交接点 A（确认布置订正） | Toast「已布置给 N 名学生」→ 自动跳「学情回收」 |
| 交接点 B（学生学会） | 跳回教师「学情回收」+ Toast「结果已回流」 |
| 闯关变式 | 选对 → 高亮绿 + Toast 鼓励 → 放行下一步；选错 → 高亮红 |
| 卡片 hover | 颜色 / 阴影变化，禁止用 scale 造成布局抖动 |
| 动效时长 | 150–400ms；尊重 `prefers-reduced-motion` |

## 7. 文案规范（转译口径）

界面文案统一走「用户语言」，与 PRD §3 名词表一致：

| 不要写（内部） | 要写（界面） |
| --- | --- |
| 交接点 A / 分发 | 确认讲评，布置订正 |
| 掌握率 79% 未达阈值 | 还有 8 人没真学会 |
| 掌握矩阵 | 全班学会情况 |
| 二次干预 | 再练一组 / 当面讲一下 |
| 长期记忆 | 学情溯源 / 我的错题本 |

**命名澄清（避免「回收」歧义）**：上游收作答的模块统一叫「**作业管理**」（不再用「作答回收」），下游收掌握结果的叫「**学情回收**」。一个是「收作业」、一个是「收会没会」，界面与文档口径一致。

语气：教师端简洁陈述（「这次答得怎么样」），学生端鼓励陪伴（「先看看，为什么错了？」「这道题，你学会了！」）。

## 11. 一句话总结

UI 以「暖纸 + 墨绿 + Loop 环」为视觉母题，教师端用侧边栏工作台承载决策密度、学生端用 Pad 闯关承载鼓励陪伴，所有界面文案完成从架构术语到用户语言的转译，让产品的双端闭环既好用、又好懂。
